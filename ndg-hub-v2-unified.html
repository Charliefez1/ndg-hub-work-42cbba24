NDG Hub v2 — Unified Spec

Source of truth. V2 PRD data model + corrected from 10-point audit + full AI integration.
What this is: The single document a developer needs to build NDG Hub v2. The V2 PRD is the data model source of truth. The previous platform map's UI/UX patterns are kept but the underlying architecture is corrected. This spec has been through two audits: (1) 10-point data model audit, (2) dual UI/UX + PM audit covering 17 fixes + UX patterns.
Key Decisions Locked

Entity Hierarchy

Organisation → Project → Delivery (UI: "Workshop") → Session → AgendaItems. Delivery is the parent. Session is a block within it.

Naming Convention

UI says "Workshops." Database says deliveries. Backend says deliveries. The sidebar page is "Workshops" but the table is deliveries.

Pricing Model

Per-workshop and per-service. NOT day rates. No day_rate or total_days on projects table. AUDIT FIX Services catalogue has fixed prices. Invoice = sum of service prices for delivered workshops.

3-Axis Placement

status on projects (7 stages). neuro_phase on deliveries (primary). kirkpatrick_level on deliveries + forms. Projects get intended_neuro_phase as metadata only.

Contacts

Tab on Client Detail only. Not a standalone page. Managed in context of their organisation.

Agenda Builder

Keep it. Sessions have timed agenda blocks (intro, activity, break, debrief, energiser). Core to delivery planning.

AI Architecture NEW

Claude API via Edge Functions (not Gemini). Sonnet for generation, Haiku for extraction. Dedicated /ai page + inline actions + contextual copilot drawer. Unified useAI hook. Persistent threads.

Design System NEW

Token-driven semantic theming. Satoshi headings, General Sans body, JetBrains Mono code. 5 accent themes (steel default). Light/dark/system. shadcn/ui base. ADHD-friendly: clear status signals, restrained motion, fast scanning. Notion/Linear-inspired.

Pricing Model
Per-workshop / per-service pricing. Your services catalogue defines fixed prices:

Workshops: Awareness, Champions, Managers, Leaders — each has a set price.
Services: Content Design, Consultancy, Policy Design, etc. — each has a set price.

When a delivery is created, you pick a service from the catalogue. The price comes with it. Invoice generation = sum up the service prices of all delivered workshops. No day rates, no time tracking needed for billing.

Architecture Rules
  • Claude API only. V1 used Lovable AI Gateway with Gemini. V2 uses Claude API via Edge Functions. Sonnet for generation, Haiku for extraction.
  • Unified useAI hook. V1 had duplicated streaming/fetch logic across AIAssistant, ProjectDetail, CreateProjectFromPlanDialog, AIChatPanel. V2 consolidates into one shared hook.
  • Rate limit: 20 calls/user/hour.
  • Cache: ai_generations table stores AI outputs with entity context for reuse.
  • Threads: ai_conversations table persists chat history. Team-shared threads optional.
  • Context injection: useAIContext() hook passes current page entity data to any AI call.

Design System

Visual language, tokens, typography, component patterns — the styling source of truth NEW
Design philosophy: Minimalist workspace UI inspired by Notion / Linear / HelloBonsai / Monday.com. Clean, low-noise, productivity-first. "ADHD-friendly" — clear status signals, restrained visual noise, fast scanning. Professional, calm, structured, modern. Softened enterprise look, not bold editorial or decorative.
Design Principles

Clarity Over Cleverness

Every element earns its place. No decorative chrome. Status is visible at a glance. Whitespace is generous. Hierarchy is enforced by size + weight, not colour alone.

ADHD-First Scanning

Key information in the top-left of every view. Status badges, not paragraphs. One primary action per screen. Colour-coded categories. No walls of text in tables.

Consistent Density

Comfortable spacing throughout — not cramped, not wasteful. Card padding 14-16px. Row height 40-44px. Gap 12-16px. Sidebar 260px. Content max-width 1200px.

Token-Driven, Not Hard-Coded

Every colour, radius, shadow, and spacing value comes from CSS custom properties or Tailwind config. Zero raw hex values in components. Theme switch changes everything.

Build Instructions for Developer
Non-negotiable design rules:

1. Token-only colours. No raw hex in any component. Every colour references a CSS custom property. This includes chart colours, SVG fills, and inline styles.

2. shadcn/ui base. All interactive components extend shadcn primitives (Button, Card, Input, Select, Dialog, Popover, Tabs, Table, Badge, Toast, Skeleton, Accordion). Don't build from scratch.

3. General Sans + Satoshi + JetBrains Mono. Load from Google Fonts or self-host. Fallback: system-ui, -apple-system, sans-serif for body; monospace for code.

4. Tailwind config extends tokens. tailwind.config.ts maps CSS custom properties to Tailwind utilities: colors: { background: 'var(--background)', surface: 'var(--surface)', ... }.

5. Status colours via utility. Create src/lib/status-colors.ts exporting getStatusColor(entity: string, status: string): string that returns the correct token. All badge/tag components use this, never hard-code status→colour mapping in JSX.

6. Theme persistence. On mount, read localStorage('ndg-theme') and localStorage('ndg-accent'). Apply to <html> element. Settings page lets user toggle light/dark/system + accent picker.

7. Responsive breakpoints. sm: 640px, md: 768px (sidebar collapses), lg: 1024px, xl: 1280px. Mobile-usable, not mobile-first.

Implementation Notes
Key decisions for the build:

Edge Functions handle all status transitions. No direct status column updates from the frontend. This ensures side-effects always fire.

Notifications table is the central hub. Every automation that needs to tell a user something inserts into notifications. Supabase Realtime pushes to connected clients. Telegram bot polls or triggers from there.

n8n runs on self-hosted instance. Hub sends webhooks to n8n. n8n calls back to Hub Edge Functions. Auth via shared secret in headers.

Scheduled jobs: pg_cron for simple checks, n8n for multi-step. "Is this invoice overdue?" = pg_cron. "Send chase email, wait 3 days, send another" = n8n.

AI automations always show preview. No silent record creation. User confirms before data is written. Exception: AI cache writes (transparent).
